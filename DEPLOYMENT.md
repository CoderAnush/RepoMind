# RepoMind Production Deployment Runbook

This runbook guides you through deploying the complete RepoMind SaaS platform on Vercel (Frontend), Render (Backend), Supabase (PostgreSQL & Auth), and Qdrant Cloud (Vector Database).

---

## 1. Prerequisites
Ensure you have accounts on:
* [GitHub](https://github.com/)
* [Supabase](https://supabase.com/)
* [Qdrant Cloud](https://qdrant.tech/)
* [Render](https://render.com/)
* [Vercel](https://vercel.com/)

---

## 2. Supabase Setup
1. Log in to [Supabase Console](https://supabase.com/).
2. Click **New Project** and name it `repomind`.
3. Wait for the database instance to provision (typically takes 1-2 minutes).
4. Navigate to **Project Settings** > **API**:
   * Copy the **Project URL** (used as `SUPABASE_URL` on Render).
   * Copy the **anon** public key (used as `SUPABASE_ANON_KEY` on Render).
   * Copy the **JWT Secret** (used as `SUPABASE_JWT_SECRET` on Render).
5. Navigate to **Project Settings** > **Database**:
   * Under **Connection String**, click **URI** and copy the Postgres URL connection string (use the transaction pooler address if deploying in an environment with high connection volume).
   * Replace the password placeholder with your actual database password.
6. Navigate to **Authentication** > **Providers** > **Email**:
   * Ensure **Enable Signup** is toggled **ON**.
   * Toggle **Confirm Email** **OFF** to allow users to sign up and immediately verify their session without checking email mailboxes.

---

## 3. Qdrant Cloud Setup
1. Log in to the [Qdrant Cloud Dashboard](https://cloud.qdrant.io/).
2. Click **Create Cluster** (Select the Free Tier Cluster which allocates 1GB RAM, 0.5 CPU, and 4GB Storage).
3. Once the cluster is active, copy the **Cluster URL** (e.g. `https://xxxx-yyy-zzz.aws.cloud.qdrant.io:6333`).
4. Click **API Keys** and generate a new key. Copy the token immediately.

---

## 4. Render Backend Deployment
1. Log in to [Render](https://render.com/).
2. Click **New +** > **Web Service**.
3. Connect your GitHub repository containing the RepoMind code.
4. Set the following parameters:
   * **Name**: `repomind-backend`
   * **Environment**: `Docker`
   * **Instance Type**: `Free` (or select a paid CPU tier to prevent instance sleep)
   * **Docker Build Context**: `backend`
   * **Dockerfile Path**: `backend/Dockerfile`
5. Click **Advanced** and add the following environment variables:
   * `DATABASE_URL` = [Your Supabase database URI connection string]
   * `SUPABASE_URL` = [Your Supabase Project URL]
   * `SUPABASE_ANON_KEY` = [Your Supabase Project Anon Key]
   * `SUPABASE_JWT_SECRET` = [Your Supabase Project JWT Secret]
   * `QDRANT_HOST` = [Your Qdrant Cluster URL]
   * `QDRANT_API_KEY` = [Your Qdrant Cluster API Key]
   * `OPENAI_API_KEY` = [Your OpenAI API Key]
   * `ANTHROPIC_API_KEY` = [Your Anthropic API Key] (Optional)
   * `LLM_PROVIDER` = `openai`
6. Click **Create Web Service**. 
7. Once deployed, note the service URL generated by Render (e.g., `https://repomind-backend.onrender.com`).

---

## 5. Vercel Frontend Deployment
1. Log in to [Vercel](https://vercel.com/).
2. Click **Add New** > **Project**.
3. Select your RepoMind GitHub repository and import it.
4. Set the following options:
   * **Root Directory**: `frontend`
   * **Framework Preset**: `Vite`
   * **Build Command**: `npm run build`
   * **Output Directory**: `dist`
5. Under **Environment Variables**, add:
   * `VITE_API_URL` = `https://repomind-backend.onrender.com/api/v1` (replace with your Render URL + `/api/v1`)
6. Click **Deploy**.
7. Vercel will build the React SPA bundle and deploy it to a global CDN network.

---

## 6. Verifying the Deployment
To confirm that your public SaaS deployment is fully functional:
1. Open your Vercel deployment URL in a browser.
2. Sign up for a new account.
3. Sign in to your account.
4. Submit a test GitHub repository URL (e.g. `https://github.com/pypa/sampleproject`).
5. Verify that:
   * The status changes to completed.
   * Documentations (README, overview, guides) generate successfully.
   * Mermaid diagrams render under the diagrams panel.
   * The conversational chat responds to questions using repository context.
